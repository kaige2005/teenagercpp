## 1. 产品概述

### 1.1 产品定位
一款面向青少年的游戏化C++编程学习平台，通过**故事化课程**、**关卡式教学**、**实时代码演练**和**可视化成果展示**，在8周内帮助零基础学员掌握C++核心语法，独立完成3个以上完整小游戏项目。

### 1.2 核心价值
- **游戏化沉浸**：故事背景"勇者的C++冒险"贯穿全程
- **即时反馈**：边学边做，每行代码立即看到效果
- **成就可视化**：积分、勋章、代码墙等激励体系
- **亲子协作**：专为家庭教学场景优化

### 1.3 教学目标矩阵

| 周次 | 主题 | C++知识点 | 游戏项目 | 能力目标 |
|------|------|-----------|----------|----------|
| 1 | 勇者启程 | 基础语法、变量、IO | 猜数字(基础) | 环境熟悉、基本交互 |
| 2 | 分支魔法 | if/else、逻辑运算 | 猜数字(进阶) | 条件判断、逻辑推理 |
| 3 | 循环奥义 | while/for、循环嵌套 | 幸运摇奖机 | 重复控制、算法思维 |
| 4 | 技能封装 | 函数、参数、返回值 | 回合制对战 | 模块化、代码复用 |
| 5 | 道具仓库 | 数组、字符串、2D数组 | 贪吃蛇(黑框) | 数据结构基础 |
| 6 | 图形魔法 | EasyX图形、动画、交互 | 弹跳小球 | 图形编程入门 |
| 7 | 角色殿堂 | 结构体、类入门 | 打砖块游戏 | 数据封装、OOP启蒙 |
| 8 | 终极挑战 | 综合项目实战 | 飞机大战/贪吃蛇(图形) | 完整项目开发 |

## 2. 用户角色

### 2.1 学员（12-16岁青少年）
- **特征**：编程零基础或初学者，对游戏/科技感兴趣
- **需求**：有趣的学习体验、即时成就感、可视化的进步
- **技能**：能够阅读基础中文、基本键盘操作、初步逻辑思维能力

### 2.2 导师（家长/一对一导师）
- **特征**：具备基础编程背景，愿意投入时间陪伴教学
- **需求**：清晰的教学指引、灵活的课程控制、学员进度跟踪
- **能力**：能够运行Dev-C++、理解C++基础语法、具备调试能力

## 3. 功能需求

### 3.1 课程管理系统

#### 3.1.1 课程编排引擎
```
需求优先级：P0（核心功能）

功能描述：
- 支持8周课程结构化编排
- 每课时可配置：知识点、代码示例、练习题、课后作业
- 支持"故事导入"文本与教学内容的关联
- 课时之间支持前置依赖（需完成前一课时才能进入下一关）

数据模型：
- Course(课程) -> Week(周) -> Lesson(课时) -> Task(任务)
- 每个课时包含：讲解内容、示例代码、练习题、项目任务
```

#### 3.1.2 进度跟踪
```
需求优先级：P0

功能描述：
- 记录学员每个课时的完成状态（未开始/进行中/已完成）
- 记录代码练习的提交次数和正确率
- 支持"复活机制"：未按时完成的作业可补交
- 导师可查看学员的学习时长、代码提交频次
```

### 3.2 游戏化激励系统

#### 3.2.1 积分系统
```
需求优先级：P1（重要功能）

功能描述：
- 完成任务 +10 积分
- 完成作业 +20 积分
- 完成项目 +50 积分
- 完成周挑战 +100 积分
- 积分可兑换虚拟奖励（后续扩展：学习特权）

数据表设计：
- 积分记录：时间、来源、数值
- 积分兑换记录：兑换项、消耗积分、兑换时间
```

#### 3.2.2 成就徽章系统
```
需求优先级：P1

功能描述：
- 8枚周章：每完成一周解锁一枚
- 最终章：集齐8枚解锁「家庭代码小达人」
- 徽章视觉风格：游戏化、可定制主题
- 徽章展示：系统内置展示墙/导出分享

徽章列表：
┌─────────┬──────────────┬──────────────┐
│ 周章    │ 名称         │ 解锁条件        │
├─────────┼──────────────┼──────────────┤
│ 第1周章 │ 新手村勇者   │ 完成第1周课程   │
│ 第2周章 │ 迷雾探索者   │ 完成第2周课程   │
│ 第3周章 │ 峡谷征服者   │ 完成第3周课程   │
│ 第4周章 │ 圣殿守护者   │ 完成第4周课程   │
│ 第5周章 │ 仓库管理员   │ 完成第5周课程   │
│ 第6周章 │ 彩虹绘师     │ 完成第6周课程   │
│ 第7周章 │ 角色设计师   │ 完成第7周课程   │
│ 第8周章 │ BUG猎手大师  │ 完成终极项目    │
└─────────┴──────────────┴──────────────┘
```

#### 3.2.3 代码展示墙
```
需求优先级：P2（增值功能）

功能描述：
- 自动收集学员的关键项目代码
- 按照课程进度展示作品时间线
- 支持代码截图保存和打印
- 支持导出为Markdown/PDF格式的"学习档案"
```

### 3.3 交互式教学环境

#### 3.3.1 代码编辑器（集成模式）
```
需求优先级：P0

功能描述：
- 提供预设代码模板，学员填空式学习
- 支持代码行级标注（重点行高亮）
- 支持"逐行运行"：标记断点、分步执行、查看变量
- 集成编译错误提示（友好化显示，如"发现BUG啦！"）

技术方案：
方案A：内嵌 Monaco/CodeMirror 编辑器（Web端）
方案B：生成 Dev-C++ 项目文件，本地编译运行
推荐：方案B，与教学计划使用环境一致
```

#### 3.3.2 可视化演示系统
```
需求优先级：P1

功能描述：
- 知识点动画演示（如：变量赋值的可视化过程）
- 代码执行流程动画（如：if分支走向、循环执行过程）
- 图形化输出结果展示（EasyX窗口的嵌入式预览）
```

### 3.4 作业与考核系统

#### 3.4.1 课后作业管理
```
需求优先级：P1

功能描述：
- 每课时配套2-3个梯度练习题
- 练习题包含：题目描述、代码框架、预期输出
- 自动判题（仅限输出比对）或导师人工批阅
- 作业提交：代码文件上传或文本粘贴
```

#### 3.4.2 周项目挑战
```
需求优先级：P0

功能描述：
- 每周设定一个完整游戏项目目标
- 提供项目需求文档（用户故事形式）
- 提供分阶段检查点（里程碑）
- 支持代码仓库管理（版本快照）
```

#### 3.4.3 BUG猎手模式
```
需求优先级：P2

功能描述：
- 提供有意埋入BUG的代码片段
- 学员需找出并修复BUG
- 导师模式：可自定义插入BUG供学员练习
- 记录发现BUG的数量作为能力指标
```

### 3.5 教学辅助工具

#### 3.5.1 导师控制台
```
需求优先级：P1

功能描述：
- 查看学员实时学习进度仪表盘
- 设置课程解锁/锁定状态
- 自定义每日作业和奖励
- 发送鼓励消息/学习提示
```

#### 3.5.2 代码规范检查器
```
需求优先级：P2

功能描述：
- 自动检查基础代码规范：缩进、命名、注释
- 温和提示：不强制阻断，仅建议改进
- 统计数据：代码风格改进趋势
```

## 4. 非功能需求

### 4.1 技术约束
| 约束项 | 要求 |
|--------|------|
| 开发环境 | Dev-C++ 5.11 + EasyX 图形库 |
| 平台兼容 | Windows 7/10/11 |
| 离线支持 | 支持完全离线运行（家庭场景网络不确定） |
| 数据存储 | 本地SQLite数据库 + 可选云端同步 |
| 部署方式 | 便携版，无需安装 |

### 4.2 用户体验要求
- **启动时间**：< 3秒
- **界面语言**：简体中文
- **字体大小**：默认14pt（可调整，照顾青少年视力）
- **配色方案**：护眼模式（深色/浅色可选）
- **无障碍**：支持键盘快捷键操作

### 4.3 安全与隐私
- 所有学习数据本地存储，不上传云端（隐私优先）
- 无账号系统，可选本地用户名
- 定期自动备份学习进度
