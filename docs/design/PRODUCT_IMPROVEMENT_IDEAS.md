## 基于设计系统的优化方向

### 方向一: 渐进式披露 (Progressive Disclosure)

**核心**: 不要一次展示所有内容

**具体优化**:
```
知识阶段:
  当前: 四个小节全部可见
  优化: 一次只展示当前小节，完成后淡入下一个小节
  
实践阶段:
  当前: 代码模板全部显示
  优化: 用TODO标记引导，逐步揭示下一步
  
主界面:
  当前: 所有课程按钮可见
  优化: 只高亮当前可学习的，其他弱化
```

### 方向二: 微交互反馈

**核心**: 每一个操作都有反馈

**待设计**:
- [ ] 按钮点击: 轻微缩放 + 声音 (可选)
- [ ] 输入正确: 绿色勾出现动画
- [ ] 输入错误: 红色抖动 + 提示
- [ ] 进度增加: XP数字滚动增长
- [ ] 解锁新内容: 金色光芒效果

### 方向三: 减少认知负荷

**核心**: 让用户思考更少

**优化点**:
1. **减少文字阅读**
   - 用图示代替文字说明
   - 用动画演示代码执行
   - 用颜色区分代码类型

2. **明确下一步**
   - 始终有清晰的"下一步"按钮
   - 当前步骤高亮，其他弱化
   - 进度可视化

3. **容错设计**
   - 错误时给出具体修复建议
   - 提供"导师放行"选项
   - 允许返回修改
