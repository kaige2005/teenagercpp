# TeenC++ 导师使用指南

本文档面向家长/导师，介绍如何指导学生使用本系统。

---

## 导师职责

1. **引导学生完成学习步骤** - 按顺序阅读左侧步骤说明
2. **解答代码疑问** - 协助理解语法和逻辑
3. **判断放行时机** - 使用"导师放行"功能
4. **查看学习进度** - 通过数据库了解学生情况

---

## 🔓 导师手动放行功能

### 何时使用

| 情况 | 建议操作 |
|------|---------|
| 学生代码正确，检查误判 | ✅ 放行，记录"代码正确，系统误判" |
| 学生理解正确，但写法略有差异 | ✅ 放行，记录"理解正确，写法变通" |
| 学生完全不会，瞎蒙的 | ❌ 不放行，继续引导 |
| 学生代码有明显语法错误 | ❌ 不放行，指出错误 |

### 放行操作步骤

1. 学生点击 **🔍 检查代码** 后，如果未通过，界面显示 **🔓 导师放行** 按钮

2. 点击按钮后弹出对话框：
```
┌─────────────────────────────────────────────┐
│ 🔓 导师手动放行                              │
├─────────────────────────────────────────────┤
│ ⚠️ 学生代码未通过自动检查，确定要放行吗？      │
│                                             │
│ 导师姓名： [_______________]                  │
│                                             │
│ 放行备注：                                   │
│ ┌─────────────────────────────────────────┐  │
│ │ 学生已理解 cout 用法，只是忘记加分号       │
│ │ 建议下次注意语句结束符                     │
│ └─────────────────────────────────────────┘  │
│                                             │
│ 请简要说明放行的原因...                      │
├─────────────────────────────────────────────┤
│ [取消]                    [✓ 确认放行]       │
└─────────────────────────────────────────────┘
```

3. 填写导师姓名和备注（可选但强烈推荐）

4. 点击 "确认放行"，系统会要求最终确认

### 放行记录

所有放行操作都会被记录到数据库：
- 放行时间
- 导师姓名
- 放行原因
- 当时的代码（可复查）

## 📊 查看学生进度

### 方式一：通过界面观察

主界面显示：
- 当前等级/经验值
- 完成课程数
- 提交次数（频繁提交可能表示遇到困难）

### 方式二：直接查看数据库

使用任何 SQLite 工具（如 [DB Browser for SQLite](https://sqlitebrowser.org/)）打开 `teen_cpp_student.db`：

#### 查看学生基本信息
```sql
SELECT * FROM StudentProgress;
```

结果示例：
```
Id  StudentName  Level  Exp  Completed  Submissions
--  -----------  -----  ---  ---------  -----------
1   小明         2      350  3          15
```

#### 查看提交历史
```sql
SELECT LessonId, SubmitTime, CheckScore, IsPassed, IsManuallyApproved, ApprovedBy
FROM Submission
ORDER BY SubmitTime DESC;
```

结果示例：
```
LessonId  SubmitTime           Score  Passed  Manual  ApprovedBy
--------  -------------------  -----  ------  ------  ----------
L02       2025-03-16 14:30:22  85     1       0       NULL
L02       2025-03-16 14:25:10  60     0       1       张老师
```

#### 统计各课尝试次数
```sql
SELECT LessonId,
       COUNT(*) as 尝试次数,
       SUM(CASE WHEN IsPassed = 1 THEN 1 ELSE 0 END) as 通过次数,
       SUM(CASE WHEN IsManuallyApproved = 1 THEN 1 ELSE 0 END) as 放行次数
FROM Submission
GROUP BY LessonId;
```

## 🎯 教学建议

### 第1课（Hello World）教学要点

1. **让学生先观察，不急着改**
   - 指出哪里是注释（绿色）
   - 哪里是代码（彩色）
   - main 函数的作用

2. **强调细节**
   - 分号 `;` 是语句结束
   - 双引号 `"` 包裹字符串
   - `<<` 是输出方向

3. **常见错误预警**
   - 中文符号（，。）vs 英文符号 (, .)
   - 漏打分号
   - 引号不匹配

### 鼓励策略

| 场景 | 回应建议 |
|------|---------|
| 一次通过 | "太棒了！一次就做对了！" |
| 多次尝试后通过 | "很好，坚持就是胜利！" |
| 需要放行 | "没关系，重要的是理解，下次注意..." |
| 完全不会 | "我们一起来看看这段代码..." |

## ⚙️ 添加新课程（高级）

如需添加新课程，编辑以下文件：

### 1. 创建课程目录结构
```
courses/
└── lesson02/
    ├── lesson.json         # 课程配置
    ├── templates/
    │   └── main.cpp        # 代码模板
    └── checks/
        └── rules.json      # 检查规则
```

### 2. 编辑课程配置 `lesson.json`

参考第一课配置，修改：
- `id`: "L02"
- `sequence`: 2
- `title`: "新标题"
- `prerequisiteId`: "L01"（前置课程）
- `steps`: 学习步骤

### 3. 设置检查规则 `checks/rules.json`

检查类型：
| Type 值 | 含义 |
|---------|------|
| 0 | Syntax - 语法检查 |
| 1 | Keyword - 关键字检查 |
| 2 | Structure - 结构检查 |
| 4 | Style - 代码风格 |

## 🐛 故障排除

### 学生说"检查按钮没反应"
- 检查是否修改了代码
- 确认 `courses/lesson01` 目录存在

### 导师放行后没有解锁下一关
- 放行后需要点击"确定"关闭对话框
- 检查数据库是否正确更新

### 忘记导师密码/权限
- 本系统暂时没有权限控制，任何人都可以点击放行
- 后续版本会添加导师密码验证
