## SKILL-01: 大纲规划 (Syllabus Planning)

**触发条件**: 开始规划新课程级别（如 L04 循环结构）

**输入**:
- NOI 大纲对应知识点
- 前置课程（如 L03 数组）
- 目标课时数

**输出**:
- `docs/syllabus/L{XX}_outline.md` - 课程大纲文档
- 知识点依赖图
- 三阶段时间分配

**检查清单**:
```markdown
□ 列出本课所有 NOI 对应知识点
□ 确定前置依赖（必须掌握的内容）
□ 设定学习目标（学生学完能做什么）
□ 分配三阶段比重（知识:实践:挑战）
□ 确定核心代码示例数量
□ 预估 XP 奖励分配
```

---

## SKILL-02: 课程设计 (Lesson Design)

**触发条件**: 大纲确定后，细化单课结构

**输入**:
- 大纲文档
- 角色设定（小极、小灵）
- UI 组件库

**输出**:
- `docs/design/L{XX}_design.md` - 详细设计文档
- Phase 结构定义
- 数据模型（LessonData/Phase/Step）

### 三阶段结构设计模板

```markdown
## 阶段一：知识讲解 (40% 课时)
- 场景故事：____场景引入问题
- 概念讲解：___个核心概念（概念卡片+代码示例）
- 知识测验：___道选择题（即时反馈）

## 阶段二：编程实践 (50% 课时)
- 代码演示：___个可运行示例
- 填空练习：___道代码填空
- 完整编程：___道独立编程题

## 阶段三：Bug猎手 (10% 课时)
- Bug案例：___个有代表性的错误
- 修复任务：逐步引导定位问题
```

## SKILL-03: 故事编排 (Story Crafting)

**触发条件**: 需要为课程设计场景和叙事

**核心原则**:
- **场景化**: 代码解决实际问题（数组→管理书架）
- **角色化**: 小极（引导）+ 小灵（提问）
- **渐进式**: 从简单场景到复杂应用

### 故事线模板

```markdown
### 开场（钩子）
小灵发现___问题，不知道如何用代码解决

### 展开（探索）
小极引导发现___概念，通过___示例演示

### 实践（应用）
学生帮助小灵完成___任务

### 高潮（挑战）
发现一段有 Bug 的代码，需要修复才能解决问题

### 收尾（成就感）
任务完成，获得 XP 奖励，解锁下一课预告
```

## SKILL-04: 概念讲解 (Concept Teaching)

**触发条件**: 设计知识阶段的内容

### 内容块标准结构

```markdown
1. **概念引入** (1-2 句话)
   - 类比生活场景
   - "就像___一样"

2. **形式化定义**  
   - 一句话精确定义
   - 配合代码示例

3. **Code Card 展示**
   ```cpp
   // 简洁示例，突出核心概念
   ```

4. **可视化辅助**（如需要）
   - 数组→书架图示
   - 循环→流程图

5. **即时测验**
   - 2-3 道选择题
   - 答错显示解释
```

---

## SKILL-05: 练习设计 (Practice Design)

**触发条件**: 设计实践阶段的练习题

### 题目梯度设计

| 难度 | 题型 | 示例 | XP |
|------|------|------|-----|
| **引导级** | 代码填空（高提示） | 给出 80% 代码，填关键字 | 10 |
| **练习级** | 代码填空（低提示） | 给出 50% 代码，填逻辑 | 20 |
| **应用级** | 独立编程（骨架代码） | 给出函数签名，自主实现 | 30 |
| **挑战级** | 独立编程（完全开放） | 仅描述需求，自主设计 | 50 |

### 每课配置建议

- 引导级 1-2 题（热身）
- 练习级 2-3 题（巩固）
- 应用级 1-2 题（运用）
- 挑战级 1 题（拔高，可选）

## SKILL-06: 挑战构建 (Challenge Building)

**触发条件**: 设计 Bug猎手模式内容

### Bug 案例设计原则

1. **典型性**: 必须是初学者常见错误
2. **教育性**: 修复后能加深对概念的理解
3. **分层提示**: 提供渐进式线索

### Bug 案例模板

```yaml
bug_id: "L04_ARRAY_INDEX_01"
type: "数组越界"
error_code: |
  int arr[5] = {1,2,3,4,5};
  cout << arr[5];  // 错误！
difficulty: "easy"
hints:
  - "观察数组声明，有效下标范围是什么？"
  - "arr[5] 访问的是第几个元素？"
  - "最后一个元素的下标应该是多少？"
explanation: "数组下标从0开始，arr[5]的有效下标是0-4"
```

## SKILL-07: 代码实现 (Code Implementation)

**触发条件**: 设计文档完成后，开发课程数据

### 开发顺序

```markdown
1. 创建 LessonData 类（继承 BaseLessonData）
2. 实现 InitializePhases() - 定义三阶段
3. 填充各 Phase 的 Steps
4. 配置代码编辑器的初始代码和预期答案
5. 设置测试用例（输入/输出对）
6. 配置 XP 奖励节点
```

### 代码规范

```csharp
// 每课独立文件，命名规范
// Lessons/L04_LoopBasicsLesson.cs

public class L04_LoopBasicsLesson : BaseLessonData
{
    public override void InitializePhases()
    {
        // Phase 1: 知识讲解
        var knowledgePhase = new Phase("知识讲解", PhaseType.Knowledge);
        // ... 
    }
}
```

## SKILL-08: 体验调优 (UX Tuning)

**触发条件**: 功能实现后，优化学习体验

### 调优检查项

```markdown
□ 故事对话是否自然？（朗读测试）
□ 代码示例是否可以直接运行？
□ 错误提示是否友好？
  （不说"编译错误"，说"好像有个小笔误"）
□ XP 获得是否有爽快感？（动画+音效）
□ 进度保存是否正常？
□ 阶段切换是否流畅？
```

## SKILL-10: 教学验证 (Pedagogy Review)

**触发条件**: 技术测试通过后，验证教学效果

### 模拟学习者测试

```markdown
□ 零基础的"我"能否理解概念讲解？
□ 完成练习后是否有成就感？
□ Bug猎手是否有"恍然大悟"的感觉？
□ 学习路径是否清晰？（不会迷失）
□ 难度曲线是否合理？（没有陡升）
```

### 专家评审（如有条件）

- 请有经验的编程教师试学
- 收集反馈并迭代

## 使用示例：L04 循环结构开发流程

```
Day 1: [SKILL-01] 大纲规划
        → 确定讲 for/while，前置是数组，课时 45分钟

Day 2: [SKILL-02] 课程设计 + [SKILL-03] 故事编排
        → 设计"自动整理书架"场景
        → 确定三阶段结构

Day 3: [SKILL-04] 概念讲解 + [SKILL-05] 练习设计
        → 写知识卡片和代码示例
        → 设计 4 道练习题

Day 4: [SKILL-06] 挑战构建 + [SKILL-07] 代码实现
        → 设计 2 个循环相关 Bug
        → 编写 LessonData 代码

Day 5: [SKILL-08] 体验调优
        → UI 微调、对话优化

Day 6: [SKILL-09] 技术测试 + [SKILL-10] 教学验证
        → 跑通测试用例
        → 模拟学习体验
```
