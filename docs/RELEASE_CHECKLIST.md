# TeenC++ 发布前强制检查清单

## 使用说明

**任何人在发布前必须完成此清单，否则发布无效。**

清单分为三个层次：
1. **自动验证** - CI/CD脚本强制执行
2. **人工检查** - 开发者必须逐项确认
3. **用户验证** - Beta测试者确认

---

## 第一层：自动验证（强制执行）

### 脚本化检查
运行发布脚本自动检查：

```powershell
# 正常发布流程
.\tools\publish.ps1 -Version "v1.2.0-beta.3"

# 会执行以下强制检查：
```

| 检查项 | 脚本 | 失败处理 |
|--------|------|----------|
| 编译 | `dotnet build` | 退出代码1，发布中止 |
| 单元测试 | `dotnet test` | 退出代码1，发布中止 |
| 文件完整性 | Copy-Item + Test-Path | 异常抛出，发布中止 |
| 课程数据 | Test-Path 课程JSON | 异常抛出，发布中止 |
| DLL齐全 | 检查x64/x86目录 | 异常抛出，发布中止 |

**关键**：这些检查在脚本中**不可跳过**，添加 `-SkipVerification` 需CTO审批。

## 第二层：人工检查（发布前24小时冻结）

代码冻结后，执行以下检查并签名：

### 2.1 完整性检查
- [ ] **x64/SQLite.Interop.dll** 存在于发布目录
- [ ] **x86/SQLite.Interop.dll** 存在于发布目录
- [ ] 课程JSON文件格式校验（无JSON语法错误）
- [ ] 模板文件存在且非空

验证命令：
```powershell
# DLL检查
Test-Path "releases\vX.X.X\x64\SQLite.Interop.dll"
Test-Path "releases\vX.X.X\x86\SQLite.Interop.dll"

# JSON校验
Get-Content "releases\vX.X.X\courses\l03\lesson.json" | ConvertFrom-Json
```

### 2.2 功能检查（人工启动验证）

**必须在干净环境（虚拟机/新目录）测试：**

| 检查项 | 验证方法 |
|--------|----------|
| 应用启动 | 双击EXE无DLL缺失错误 |
| 数据库初始化 | 无异常弹窗 |
| 主界面显示 | 地图、按钮正常 |
| L01加载 | 代码编辑器可见 |
| L02加载 | 代码编辑器可见 |
| L03加载 | 阶段选择界面显示 |
| 代码检查 | 任一代码点击"检查"有响应 |
| 项目生成 | 能生成.dev文件 |

### 3.2 准入门槛

| 阈值 | 要求 |
|------|------|
| 启动成功率 | 100% (>=3个不同环境) |
| 课程加载成功率 | 100% |

## 本次v1.2.0教训沉淀

### 问题根源
| 层级 | 问题 | 根本原因 |
|------|------|----------|
| 自动验证 | 无 | 脚本未加入x64/x86检查 |
| 人工检查 | 跳过 | 直接复制文件，未在干净环境测试 |
| 用户验证 | 首次暴露 | DLL缺失问题本应在发布前发现 |

### 改进措施（已实施）

1. **发布脚本增强** (`publish.ps1`)
   - 强制检查 x64\SQLite.Interop.dll
   - 强制检查 x86\SQLite.Interop.dll
   - 验证EXE PE头完整性

2. **人工检查清单** (本文档)
   - 明确列出DLL检查项
   - 强制干净环境测试

3. **CI/CD集成** (后续)
   - 每次构建生成完整发布包
   - 验证发布包能启动
   - 自动化Smoke Test
