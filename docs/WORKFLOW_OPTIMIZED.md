# TeenC++ 教学系统 - 优化后的提测流程

## 方案概述
三级提测标准，根据版本类型选择不同流程：

| 版本类型 | 标准 | 使用场景 |
|----------|------|----------|
| **Alpha** | 自测通过 | 个人开发验证 |
| **Beta** | 轻量清单 | 迭代中用户测试 |
| **RC** | 标准模式 | 正式发布前验证 |
| **正式版** | 完整流程 | 里程碑版本 |

---

## 详细标准

### Alpha（自测）
- [ ] 程序能启动
- [ ] 主要功能可运行
- [ ] 无明显崩溃
- **无需文档**，快速迭代

### Beta（轻量清单）← 当前使用
**前置要求**：
- [ ] 单元测试通过
- [ ] 自测检查清单（5-10项核心功能）

**交付物**：
- 测试清单（标记通过/失败）
- 版本号（x.y.z-beta.n）

**用户测试反馈**：问题描述 → 开发修复 → 快速迭代

### RC（标准模式）
**前置要求**：
- [ ] 单元测试覆盖率 ≥ 80%（核心模块）
- [ ] 功能测试通过
- [ ] 回归测试通过

**交付物**：
- 测试报告
- 发布说明
- 已知问题清单

### 正式版（完整流程）
**前置要求**：
- [ ] 全部测试覆盖要求
- [ ] 代码审查通过
- [ ] 文档完整（PRD、架构、API、用户手册）
- [ ] 评审会议通过
